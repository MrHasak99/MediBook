# Plan: Hide File Names from URLs (Clean URLs)

The goal is to implement clean URLs (e.g., `/doctors` instead of `/doctors.html`) across the application to improve UX and SEO.

## User Review Required

> [!IMPORTANT]
> This plan will modify multiple files in the `client/` directory, specifically removing `.html` extensions from internal links. This assumes the hosting provider (Netlify) is configured to handle these clean URLs.

- Are there any other specific URL patterns that should be hidden or renamed?
- Should I also handle potential broken links if users have bookmarked the old `.html` URLs? (Netlify handles this if we use 301 redirects, but usually "pretty URLs" just works side-by-side or redirects automatically).

## Proposed Changes

### 1. Configure Netlify for Pretty URLs

- **File:** `netlify.toml`
- **Action:** Add asset optimization settings to automatically handle clean URLs and redirect `.html` requests to their non-extension counterparts.
- **Code:**
  ```toml
  [build.processing]
    skip_processing = false
  [build.processing.html]
    pretty_urls = true
  ```
